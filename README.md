# Lab 3 — Защита от конфликтов при одновременном копировании

Программа secure_copy обрабатывает N файлов параллельно через 3 потока
с мьютексной синхронизацией и логированием в log.txt.

## СБОРКА

make

## ЗАПУСК

./secure_copy file1.txt file2.txt file3.txt output_dir/ K

  file1.txt ...  — входные файлы (любое количество)
  output_dir/    — выходная директория (создаётся автоматически)
  K              — ключ шифрования (один символ)

## ТЕСТЫ

make test           — запуск на 5 файлах, вывод outdir/ и log.txt
make test_roundtrip — проверка обратимости шифрования

## СТРУКТУРА

deps/libcaesar.so   — XOR-библиотека из Lab 1
src/main.cpp        — точка входа, запуск 3 потоков
src/worker.hpp      — логика воркера: читать→шифровать→писать
src/file_queue.hpp  — потокобезопасная очередь файлов + счётчик
src/logger.hpp      — запись в log.txt (append)
src/mutex_utils.hpp — эмуляция timedlock для macOS

## СИНХРОНИЗАЦИЯ

Один мьютекс на всю программу.
Таймаут ожидания — 5 секунд, при превышении выводится предупреждение.
Мьютекс захватывается только на короткие операции:
  next_file()        — взять имя следующего файла
  increment_copied() — счётчик++
  log()              — запись строки в log.txt
